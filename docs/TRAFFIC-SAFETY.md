# 流量安全说明

## 问题背景

R2 直链大视频（如 200MB+）在未做防护时，可能导致单次访问消耗 1GB+ 流量，短时间内耗尽配额。

## 已实施的防护措施

### 1. 列表/推荐区缩略图：静态占位符

- **禁止**使用 `<video>` 加载视频获取首帧作为缩略图
- 使用 `VideoPlaceholder` 组件：仅显示播放图标，**零网络请求**
- 原因：`preload="metadata"` 对 moov 在文件末尾的 MP4 无效，仍会下载完整文件

### 2. 主播放器：按需加载

- **禁止** `autoPlay`：用户必须点击播放按钮才会开始加载
- **使用** `preload="none"`：进入页面时不加载任何视频数据，仅用户点击播放后开始
- **禁止**将视频 URL 作为 `poster`：避免以海报名义触发视频下载

### 3. 暂停部署

- `npm run deploy:pause`：部署极简维护页，停止所有应用逻辑和视频加载
- 恢复：`npm run deploy`

## 禁止事项

1. **禁止**在列表/卡片中使用 `<video src={videoUrl}>` 或任何会加载完整视频的组件
2. **禁止**主播放器使用 `autoPlay` 或 `preload="auto"`
3. **禁止**将 R2 视频 URL 用作 `poster`、`background-image` 等会触发请求的属性

## 恢复站点前检查清单

- [ ] 确认无 VideoThumbnail 或类似“加载视频取首帧”的逻辑
- [ ] 确认主播放器无 autoPlay
- [ ] 确认主播放器 preload="none"
- [ ] 本地测试：打开首页 → 网络面板应无视频请求；进入视频页 → 未点击播放前应无大流量
